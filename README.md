# 폴더별 PDF 통합 앱

Windows에서 루트 폴더를 선택하면 해당 폴더와 모든 하위 폴더를 검색해, PDF 파일이 들어있는 각 폴더마다 하나의 통합 PDF를 생성하는 Python Tkinter 앱입니다.

## 설치

Python 3.12 이상을 권장합니다.

```powershell
python -m pip install -r requirements.txt
```

## 실행

```powershell
python merge_pdfs_app.py
```

## 사용 방법

1. `찾아보기` 버튼으로 PDF를 검색할 루트 폴더를 선택합니다.
2. `미리보기/스캔`을 눌러 병합 대상 폴더와 출력 파일명을 확인합니다.
3. 필요하면 `기존 통합 PDF 덮어쓰기`를 선택합니다.
4. `병합 실행`을 눌러 각 폴더별 통합 PDF를 생성합니다.
5. 화면 하단 로그에서 성공, 건너뜀, 실패 결과를 확인합니다.

## 출력 파일명 규칙

결과 PDF는 PDF가 들어있는 원본 폴더 안에 생성됩니다.

기본 파일명은 선택한 루트 폴더명부터 PDF가 들어있는 폴더명까지의 경로 조각을 `_`로 연결하고, 마지막에 `_merged.pdf`를 붙입니다.

예시:

- 선택 루트: `A`
- PDF 폴더: `A\B\C`
- 출력 파일: `A\B\C\A_B_C_merged.pdf`

`data\分野別難度別問題`를 선택하고 대상 폴더가 `割合と比\２１～４０％`이면 출력 파일명은 `分野別難度別問題_割合と比_２１～４０％_merged.pdf`입니다.

## 병합 규칙

- 선택한 루트 폴더 자신과 모든 하위 폴더를 검색합니다.
- 각 폴더에 직접 들어있는 PDF만 해당 폴더의 병합 대상입니다.
- 하위 폴더의 PDF는 부모 폴더 결과물에 포함하지 않습니다.
- 앱이 생성한 `*_merged.pdf` 파일은 다음 실행의 입력 PDF에서 제외됩니다.
- 입력 PDF는 파일명 오름차순으로 병합됩니다.
- 원본 PDF 파일은 수정하거나 삭제하지 않습니다.

## 덮어쓰기 옵션

같은 이름의 통합 PDF가 이미 있을 때:

- 덮어쓰기가 꺼져 있으면 해당 폴더는 건너뜁니다.
- 덮어쓰기가 켜져 있으면 기존 결과 PDF를 새 결과로 교체합니다.

## 문제 해결

- `pypdf` 모듈 오류가 나면 `python -m pip install -r requirements.txt`를 다시 실행하세요.
- 폴더 선택 후 대상이 없다고 나오면, 선택한 폴더 또는 하위 폴더에 직접 들어있는 `.pdf` 파일이 있는지 확인하세요.
- 특정 폴더만 실패하면 로그에 표시된 PDF 파일이 암호화되었거나 손상되었을 가능성이 있습니다.
- 결과 파일이 생성되지 않으면 해당 폴더에 쓰기 권한이 있는지 확인하세요.
